# Story 1.3 — AI Generation API Route

**Epic:** AI Content Generation Module
**Status:** InProgress
**Priority:** High
**Complexity:** M (5 points)
**Agent:** @dev

---

## Description

Implement the main API endpoint that orchestrates the full AI content generation pipeline: receives theme + optional URL, scrapes reference content if provided, calls Gemini 2.5 Flash, validates the output, and saves as draft in Supabase.

## Acceptance Criteria

### AC-1: Orchestration
- [x] Given `POST /api/admin/ai/generate` with `{ theme }`, when authenticated, then it calls Gemini with the built prompt and returns a generated post
- [x] Given `{ theme, url }`, when URL is provided, then it first scrapes the URL, then includes scraped content in the prompt
- [x] Given scraping fails, then generation proceeds without reference content (no blocking)
- [x] Given the full pipeline, then total execution time is < 30 seconds

### AC-2: Validation & Sanitization
- [x] Given the Gemini response, then blocks are validated against Zod schemas
- [x] Given the Gemini response, then HTML in text blocks is sanitized with DOMPurify
- [x] Given invalid blocks, then they are filtered out and the remaining valid blocks are kept
- [x] Given the post metadata, then `slug` is generated from title using slugify (existing util)
- [x] Given the post metadata, then `word_count` and `reading_time_minutes` are calculated

### AC-3: Save to Supabase
- [x] Given a valid generated post, then it is saved to `blog_posts` with `status='draft'`
- [x] Given the blocks array, then each block is saved to `blog_post_blocks` with correct `order`
- [x] Given the save is successful, then the response includes `postId` for preview navigation
- [x] Given the seo_config and geo_config, then they are saved as JSONB on the post record

### AC-4: Error Handling
- [x] Given an unauthenticated request, then return 401
- [x] Given `theme` is missing or < 10 characters, then return 400 with validation error
- [x] Given Gemini API fails, then return 500 with clear error message and suggestion to retry
- [x] Given Gemini returns unparseable JSON, then return 500 with error details

### AC-5: Security
- [x] Given the endpoint, then JWT auth is required (using existing `verifyAdminAuth`)
- [x] Given the implementation, then `GEMINI_API_KEY` is never exposed in the response
- [x] Given the theme input, then it is trimmed and validated (max 500 chars)

## Technical Notes
- Use existing Supabase service role client for database writes
- Use existing `verifyAdminAuth()` for JWT verification
- Use existing `slugify` for slug generation
- Use existing `calculateMetrics()` or equivalent from block-helpers for word count / reading time
- The endpoint should stream a loading state or return when complete (v1: wait for complete response)
- Consider using Gemini's `responseSchema` parameter to enforce JSON structure
