## Story

**As a** content editor,
**I want** a visual editor where I can add, reorder, and remove blocks with drag-and-drop,
**so that** I have granular control over the post structure.

## Acceptance Criteria

1. `BlockEditor.tsx` implemented as the main editor container with state managed via `useReducer` — actions: `ADD_BLOCK`, `REMOVE_BLOCK`, `UPDATE_BLOCK`, `REORDER_BLOCKS`, `DUPLICATE_BLOCK`, `SET_BLOCKS`
2. `BlockWrapper.tsx` implemented wrapping each block with: drag handle (grip icon), block type indicator, context menu (duplicate, move up, move down, delete), delete button
3. Drag-and-drop functional via `@dnd-kit/core` + `@dnd-kit/sortable` with visual feedback (drop indicator line, opacity change on dragged item)
4. `BlockToolbar.tsx` implemented with icons for the 8 most-used blocks (heading, paragraph, image, quote, list, divider, faq, embed) + "More" button showing all 16 types
5. Insert toolbar appears between blocks on hover (shows "+" button) and is always visible at the end of the block list
6. Clicking "+" inserts a new block at the correct position (between the two blocks where "+" was clicked)
7. Auto-save functional with 30-second debounce: calls `POST /api/admin/posts/[id]/auto-save`, displays visual status indicator (Saved / Saving... / Error)
8. Slash command (`/`) in empty block opens `SlashCommand.tsx` with searchable block type selector
9. Keyboard shortcuts functional: `Ctrl+S` (save draft), `Ctrl+Shift+P` (toggle preview), `Ctrl+D` (duplicate current block), `Delete` on empty block (remove block)
10. Editor accepts initial `blocks[]` array (for edit mode) and emits `onChange(blocks)` for parent form integration

## Integration Verification
- IV1: Editor loads in < 2s with 20+ blocks
- IV2: Drag-and-drop does not cause unnecessary re-renders (use React.memo on BlockWrapper)
- IV3: Auto-save calls the API endpoint from Story 1.2 correctly

### Key Technical Details
- **dnd-kit**: Use `DndContext` with `closestCenter` collision detection. `SortableContext` with `verticalListSortingStrategy`. Each block uses `useSortable` hook.
- **Block IDs**: Use `crypto.randomUUID()` for new blocks. Existing blocks from DB keep their UUID.
- **State shape**: `blocks: ContentBlock[]` sorted by `order`. Reducer auto-recalculates `order` (0, 1, 2, ...) after every add/remove/reorder.
- **Auto-save**: Only fires when `postId` is available (edit mode). New posts must be saved first via explicit Save Draft.
- **Existing UI components**: Use `Button`, `Input`, `Select` etc. from `components/ui/` (Radix-based). Use `lucide-react` for icons.
- **Placeholder editors**: Story 1.4 will replace these with real per-type editors. Use a simple textarea that serializes to `data: { raw: string }` for now.
