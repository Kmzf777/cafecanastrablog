# Story 1.2 — URL Scraper Module

**Epic:** AI Content Generation Module
**Status:** InReview
**Priority:** High
**Complexity:** S (3 points)
**Agent:** @dev

---

## Description

Implement the URL scraping module that extracts clean text content from blog/news URLs to use as reference context for AI content generation. Uses cheerio for lightweight HTML parsing — no Playwright needed.

## Acceptance Criteria

### AC-1: Scraper Function
- [x] Given a valid URL to a blog/news article, when `scrapeUrl(url)` is called, then it returns `{ title, description, content }` with clean text
- [x] Given the scraped HTML, then `<nav>`, `<footer>`, `<aside>`, `<script>`, `<style>`, ad elements, and cookie banners are removed
- [x] Given the scraped HTML, then content is extracted from `<article>`, `<main>`, or `<body>` (in priority order)
- [x] Given the scraped content, then the output is limited to 10,000 characters maximum
- [x] Given the scraped HTML, then the page `<title>` and `<meta name="description">` are extracted

### AC-2: Error Handling
- [x] Given an unreachable URL, when scraping is attempted, then it returns `{ success: false, error: "URL inaccessible" }` within 15 seconds
- [x] Given a URL that times out, when 15 seconds elapse, then the request is aborted and returns a friendly error
- [x] Given a URL with no extractable content, then it returns `{ success: false, error: "No content found" }`
- [x] Given an invalid URL format, then it returns `{ success: false, error: "Invalid URL" }`

### AC-3: API Route
- [x] Given `POST /api/admin/ai/scrape` with `{ url }`, when the user is authenticated (JWT), then it executes the scraper and returns the result
- [x] Given an unauthenticated request, then it returns 401
- [x] Given the response, then it matches the type: `{ success: boolean, title?: string, description?: string, content?: string, error?: string }`

## Scope

### IN
- `src/lib/ai/scraper.ts` — Core scraping logic with cheerio
- `src/app/api/admin/ai/scrape/route.ts` — API endpoint
- Install `cheerio` dependency

### OUT
- Playwright-based scraping (out of scope for v1)
- JavaScript-rendered content (cheerio only parses static HTML)
- Scraping behind authentication/paywalls

## Dependencies
- None (independent from Story 1.1)

## Technical Notes
- Use `fetch` with `AbortController` for timeout (15s)
- Set a reasonable `User-Agent` header to avoid blocks
- cheerio API is similar to jQuery — use `$('article').text()` pattern
- Strip excessive whitespace and normalize line breaks in output
- The scrape endpoint is called by the generate endpoint (Story 1.3), but also usable standalone
