## Story

**As a** blog visitor,
**I want** to see posts rendered with rich visual blocks and a table of contents,
**so that** I have a superior reading experience.

## Acceptance Criteria

1. `BlockRenderer.tsx` implemented as a switch/map component that renders the correct display component for each block type
2. 16 rendering components created in `components/blog/blocks/`:
   - `HeadingBlock` — `<h1-h6>` with anchor link (clickable `#` icon on hover)
   - `ParagraphBlock` — `<p>` rendering sanitized HTML (bold, italic, underline, links, highlight)
   - `ImageBlock` — `<figure>` with Next.js `<Image>`, `<figcaption>`, `alt` attribute, lazy loading
   - `GalleryBlock` — Responsive grid/carousel/masonry layout based on `layout` setting
   - `QuoteBlock` — `<blockquote>` with `<cite>` linking to `source_url`
   - `ListBlock` — `<ul>` or `<ol>` with rendered items
   - `CodeBlock` — `<pre><code>` with syntax highlighting via `prism-react-renderer`
   - `EmbedBlock` — Responsive iframe for YouTube/Instagram/Twitter/TikTok
   - `DividerBlock` — Styled `<hr>` based on style setting
   - `CalloutBlock` — `<aside>` with icon and colored background per type
   - `TableBlock` — `<table>` with responsive horizontal scroll wrapper, `<caption>`
   - `FaqBlock` — Interactive accordion with expand/collapse animation
   - `CtaBlock` — Styled CTA banner with button
   - `VideoBlock` — `<video>` tag or embed iframe based on URL
   - `AccordionBlock` — Collapsible sections with smooth animation
   - `ProductBlock` — Product card with image, name, price, link button
3. `TableOfContents.tsx` auto-generated from `heading` blocks, displayed as sticky sidebar on desktop (right side), collapsible floating menu on mobile
4. Reading time displayed in post header (calculated from text blocks)
5. Blog post page (`/blog/[slug]/page.tsx`) refactored to fetch and render blocks via `BlockRenderer` instead of `ReactMarkdown`
6. Blog listing page (`/blog/page.tsx`) updated to use excerpt and featured image from new post fields
7. All rendering components use Tailwind CSS, consistent with existing blog design (`prose` classes where appropriate)
8. Install `prism-react-renderer` for code block syntax highlighting

### Key Technical Details
- **HTML in paragraphs**: Content from TipTap is saved as sanitized HTML. Use `dangerouslySetInnerHTML` with `prose` classes. Content was sanitized server-side on save (Story 1.2).
- **Next.js Image**: Use for `ImageBlock`. Set `width` and `height` from block data, or use `fill` with container. `loading="lazy"` by default.
- **prism-react-renderer**: Import `Highlight` and a theme (e.g., `themes.nightOwl`). Render with language from block data.
- **TOC scroll tracking**: Use `IntersectionObserver` to detect which heading is in viewport, highlight corresponding TOC item.
- **Responsive embeds**: Wrap iframes in `<div className="relative pb-[56.25%] h-0">` for 16:9 aspect ratio.
- **FAQ/Accordion animation**: Use CSS `max-height` transition or Radix `Collapsible` component.
- **Block settings**: Each block's `settings.width` (narrow/normal/wide/full) should map to max-width classes. `settings.alignment` maps to text-align.
- **Backward compatibility**: Migrated posts have a single `paragraph` block with Markdown content. Since it's stored as plain text (not HTML), render it with `ReactMarkdown` as fallback when block data doesn't contain HTML tags. OR: the migration script in Story 1.1 should preserve the Markdown as-is, and ParagraphBlock detects and renders Markdown when no HTML tags present.
