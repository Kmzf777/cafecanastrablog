# Story 1.5 — Preview Page & Dashboard Integration

**Epic:** AI Content Generation Module
**Status:** InProgress
**Priority:** High
**Complexity:** M (5 points)
**Agent:** @dev

---

## Description

Create the preview page that displays the AI-generated post before publishing, with actions to Edit, Approve, Save as Draft, or Discard. Also integrate a "Generate with AI" button into the existing AdminDashboard.

## Acceptance Criteria

### AC-1: Preview Page
- [x] Given the user is redirected to `/admin/posts/generate/preview?postId={id}`, then the page fetches the draft post from Supabase and renders it
- [x] Given the post data, then it renders all blocks using the existing blog post rendering components (same as public `/blog/[slug]`)
- [x] Given the post data, then it displays metadata above the preview: title, category, tags, excerpt
- [x] Given the post data, then it displays the SEO score (using existing SeoScorer)
- [x] Given the post data, then it displays a badge/label indicating "AI Generated — Draft"

### AC-2: Action Buttons
- [x] Given the preview page, then 4 action buttons are displayed: "Editar", "Aprovar e Publicar", "Salvar como Rascunho", "Descartar"
- [x] Given user clicks "Editar", then they are redirected to `/admin/posts/{postId}/edit` (existing block editor with content pre-loaded)
- [x] Given user clicks "Aprovar e Publicar", then the post status is updated to `published` with `published_at = now()`, and user is redirected to `/admin` with success toast
- [x] Given user clicks "Salvar como Rascunho", then the post remains as `draft` and user is redirected to `/admin` with confirmation toast
- [x] Given user clicks "Descartar", then a confirmation dialog appears; on confirm, the draft is deleted from Supabase and user returns to `/admin/posts/generate`

### AC-3: Dashboard Integration
- [x] Given the AdminDashboard, then a "Gerar com IA" button is displayed next to the existing "Novo Post" button
- [x] Given the "Gerar com IA" button, then it has a sparkles/wand icon to differentiate from manual creation
- [x] Given user clicks "Gerar com IA", then they are navigated to `/admin/posts/generate`

### AC-4: Error States
- [x] Given an invalid or non-existent `postId` param, then an error message is shown with a link back to generate page
- [x] Given the publish action fails, then an error toast is shown and the post remains as draft
- [x] Given the delete action fails, then an error toast is shown

### AC-5: Authentication
- [x] Given the preview page, then it requires admin authentication
- [x] Given an unauthenticated user, then they are redirected to `/admin/login`

## Dependencies

- **Story 1.4** — Generate page (navigation source)

## Technical Notes
- Reuse the existing public blog post renderer for consistent preview
- For the "Aprovar e Publicar" action, use existing `PUT /api/admin/posts/[id]` with `status: 'published'`
- For delete, use existing delete endpoint or add a simple DELETE to `/api/admin/posts/[id]`
- The preview should look as close as possible to the published blog post
- Use `useSearchParams` to read `postId` from URL
- Confirmation dialog for discard: use existing AlertDialog from shadcn/ui or simple `window.confirm`
- Action buttons should be sticky at the top or bottom for easy access while scrolling the preview
