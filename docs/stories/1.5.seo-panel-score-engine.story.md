## Story

**As a** content editor,
**I want** to see the SEO score in real-time while editing,
**so that** I can optimize every post before publishing.

## Acceptance Criteria

1. `seo-scorer.ts` implemented in `lib/seo/` with 14 checks as defined in PRD FR10:
   - Focus keyword in title (10 pts)
   - Focus keyword in meta description (10 pts)
   - Focus keyword in first paragraph (8 pts)
   - Keyword density 1-2% (8 pts)
   - Meta description length 120-160 chars (8 pts)
   - Meta title length 30-60 chars (7 pts)
   - Heading hierarchy H1>H2>H3 no gaps (7 pts)
   - Internal links >= 2 (7 pts)
   - External links >= 1 (5 pts)
   - All images have alt text (8 pts)
   - Content length >= 800 words (7 pts)
   - Slug optimized: keyword present, max 5 words (5 pts)
   - Reading time calculated and available (5 pts)
   - Subheading distribution: H2/H3 every ~300 words (5 pts)
2. Score engine is pure client-side function: `calculateSeoScore(blocks: ContentBlock[], seoConfig: SeoConfig, slug: string): SeoScoreResult` — no API calls
3. `SeoPanel.tsx` implemented as a tab in the editor sidebar showing:
   - Score gauge (0-100) with color: green (90+), yellow (70-89), orange (50-69), red (<50)
   - List of 14 individual checks with status icon (checkmark/warning/X) and descriptive message
4. SEO config fields editable in the panel: `meta_title` (with 60 char counter), `meta_description` (with 160 char counter), `focus_keyword`, `secondary_keywords` (up to 5, comma-separated), `canonical_url`, `og_title`, `og_description`, `og_image`, `twitter_card_type` dropdown, `noindex` toggle, `nofollow` toggle
5. SERP preview showing how the result will appear in Google: title (blue link), URL (green), description (gray) — updates in real-time
6. Score recalculates in real-time on every content or SEO config change (debounced 500ms)
7. `SeoScoreGauge.tsx` component with circular or bar gauge visualization
8. `SeoCheckItem.tsx` component for individual check display
9. SEO score saved to post's `seo_score` field when post is saved

## Integration Verification
- IV1: Score calculation is entirely client-side, zero API calls
- IV2: Recalculation completes in < 50ms for a post with 30 blocks
- IV3: SEO config data saved correctly to `seo_config` JSONB field via API

### Key Technical Details
- **Pure client-side**: The SEO scorer must NOT make any API calls. All analysis is done on the blocks array and config in memory.
- **Text extraction**: Blocks that contain text: `heading.text`, `paragraph.text` (HTML — strip tags first), `quote.text`, `callout.text`, `list.items[]`, `faq.items[].question + .answer`, `accordion.items[].title + .content`, `cta.text`, `product.description`
- **Link extraction**: Parse `<a href>` tags from paragraph, callout, list items. Internal link = starts with `/` or contains `cafecanastra.com`. External = all others.
- **Heading hierarchy check**: Extract heading levels in order, verify no gaps (e.g., H1 then H3 without H2 = fail)
- **Keyword density**: `(keyword occurrences / total words) * 100`. Target: 1-2%.
- **Score weights**: Total possible = 100. Each check has a weight (see AC1). Partial scores awarded for "close" values (e.g., 80% of points if meta desc is 100-119 chars).
- **Debounce**: Use 500ms debounce on the score calculation to avoid recalculating on every keystroke.
- **Existing UI**: Use Tailwind for styling. Radix `Switch` for toggles. Radix `Select` for dropdowns.
