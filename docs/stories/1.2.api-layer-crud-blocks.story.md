## Story

**As a** blog administrator,
**I want** to create and edit posts with blocks via API,
**so that** the editor can save and retrieve structured content.

## Acceptance Criteria

1. `POST /api/admin/posts` accepts payload with `blocks[]`, `seo_config`, `geo_config` and creates post + blocks atomically (single transaction via Supabase)
2. `PUT /api/admin/posts/[id]` updates post metadata and synchronizes blocks: deletes orphaned blocks, updates existing ones, inserts new ones — all atomically
3. `GET /api/admin/posts/[id]` returns post with blocks ordered by `order` field, including `seo_config` and `geo_config`
4. `POST /api/admin/posts/[id]/blocks/reorder` accepts array of `{id, order}` and updates order for multiple blocks in one request
5. `POST /api/admin/posts/[id]/auto-save` saves partial post data without full validation (accepts incomplete blocks), stores with status `draft`
6. Server-side Zod validation applied to all block data using schemas from Story 1.1
7. HTML sanitization via DOMPurify (or equivalent server-side) applied to all rich text fields in block data (`text` in paragraph, quote, callout, list items, accordion content, FAQ answers)
8. All admin API endpoints protected by existing JWT auth middleware

### Key Technical Details
- **Supabase client**: Use `createClient` with service role key (server-side) for write operations
- **Auth**: Existing `verifyAdminAuth()` from `lib/admin-auth.ts` — call at start of each handler
- **Transaction safety**: Supabase JS doesn't have native transactions. Use delete-all-then-insert pattern for block sync on PUT. The CASCADE on `post_id` FK handles cleanup on post delete.
- **Reading time calculation**: Count words in all text blocks (paragraph, heading, quote, callout, list items, FAQ answers, accordion content), divide by 200 (words per minute)
- **Word count**: Sum of all text content words across all blocks
- **Sanitization targets**: `paragraph.text`, `quote.text`, `callout.text`, `list.items[]`, `faq.items[].answer`, `accordion.items[].content`
