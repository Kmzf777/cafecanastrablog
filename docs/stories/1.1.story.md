# Story 1.1 — Gemini Client & Prompt Builder

**Epic:** AI Content Generation Module
**Status:** InProgress
**Priority:** High
**Complexity:** M (5 points)
**Agent:** @dev

---

## Description

Setup the core AI infrastructure: Gemini client configuration and the prompt builder that generates SEO/GEO-optimized blog posts. This is the foundation that all other stories depend on.

## Acceptance Criteria

### AC-1: Gemini Client
- [x] Given `GEMINI_API_KEY` is set in `.env.local`, when the client is initialized, then it connects to Gemini 2.5 Flash successfully
- [x] Given the client is configured, when a request is made, then it uses `responseMimeType: "application/json"` for structured output
- [x] Given the client is configured, then `temperature` is set to `0.7`
- [x] Given `GEMINI_API_KEY` is missing, when the API route is called, then it returns a clear error message

### AC-2: Prompt Builder
- [x] Given a theme string, when `buildPrompt(theme)` is called, then it returns a complete prompt with SEO/GEO rules
- [x] Given a theme + reference content, when `buildPrompt(theme, reference)` is called, then it includes the reference as context with instructions to rewrite (not copy)
- [x] Given the prompt output, then it instructs Gemini to generate: title (50-70 chars), excerpt (120-160 chars), slug, category (from valid enum), tags (3-5), seo_config, geo_config
- [x] Given the prompt output, then it instructs Gemini to generate 8-15 blocks including: heading H1, paragraph intro, at least 3x heading H2, paragraphs, callout, faq (3-5 items), cta
- [x] Given the prompt output, then SEO rules are included: focus keyword in H1/first paragraph/H2/meta_description, keyword density 1-2%, 800-1500 words
- [x] Given the prompt output, then GEO rules are included: Serra da Canastra/Minas Gerais mentions, semantic entities, citable structure
- [x] Given the prompt output, then the Cafe Canastra brand voice is enforced (warm, knowledgeable, artisanal)

### AC-3: Response Parser
- [x] Given valid JSON from Gemini, when `parseResponse(json)` is called, then it returns typed `{ post, blocks }` matching existing interfaces
- [x] Given each block in the response, then it is validated against the existing Zod block schemas
- [x] Given invalid blocks in the response, then they are filtered out (not crash the entire response)
- [x] Given the parsed response, then HTML content in paragraph/callout/faq blocks is sanitized with DOMPurify

## Scope

### IN
- `lib/ai/gemini-client.ts` — Gemini SDK initialization and configuration
- `lib/ai/prompt-builder.ts` — Prompt construction with SEO/GEO rules
- `lib/ai/response-parser.ts` — Parse, validate, sanitize Gemini output
- Install `@google/genai` dependency

### OUT

- UI pages (Story 1.4, 1.5)
