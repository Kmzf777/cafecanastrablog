## Story
**As a** responsável pelo SEO e GEO do Café Canastra,
**I want** que a homepage possua schema markup JSON-LD com `Organization`, `LocalBusiness`, `WebSite` com `SearchAction` e `FAQPage`,
**so that** o Google exiba rich snippets nos resultados de busca e as IAs generativas (ChatGPT, Gemini, Perplexity) identifiquem o Café Canastra como uma fonte autoritativa sobre café especial da Serra da Canastra.

## Acceptance Criteria

1. A homepage possui um bloco `<script type="application/ld+json">` com schema `Organization` contendo:
   - `name`: "Café Canastra"
   - `url`: URL do site
   - `logo`: URL da logo
   - `description`: descrição da empresa
   - `sameAs`: array com URLs das redes sociais (Instagram, Facebook, YouTube)

2. A homepage possui schema `LocalBusiness` contendo:
   - `@type`: ["LocalBusiness", "FoodEstablishment"]
   - `name`, `description`, `url`
   - `address` com `addressLocality` (Serra da Canastra), `addressRegion` (MG), `addressCountry` (BR)
   - `priceRange`: "$$"
   - `servesCuisine`: "Café Especial"

3. A homepage possui schema `WebSite` com `SearchAction` do tipo `SearchAction`:
   ```json
   {
     "@type": "WebSite",
     "url": "https://cafecanastra.com",
     "potentialAction": {
       "@type": "SearchAction",
       "target": "https://cafecanastra.com/blog?q={search_term_string}",
       "query-input": "required name=search_term_string"
     }
   }
   ```

4. A homepage possui schema `FAQPage` com mínimo de 8 perguntas e respostas sobre:
   - O que é café especial?
   - O que torna o café da Serra da Canastra especial?
   - Quais produtos o Café Canastra oferece?
   - Como é feita a torra artesanal?
   - Como comprar café Canastra?
   - Qual a diferença entre grão e moído?
   - O que é pontuação SCA?
   - Por que a altitude influencia o sabor do café?

5. Todos os schemas validam sem erros no [Google Rich Results Test](https://search.google.com/test/rich-results).

6. Os schemas são implementados como componentes React no arquivo de layout ou página, usando `<script>` tags com JSON-LD — não como texto inline ou bibliotecas externas.

7. Os dados do schema (nome, endereço, redes sociais) são configuráveis via constantes em um arquivo de configuração, sem hardcode espalhado.

## Dev Notes

### Implementação JSON-LD no Next.js
No Next.js App Router, schemas são adicionados via `<script>` tag no componente de página:

```tsx
// components/schema-org.tsx
export function SchemaOrg({ schema }: { schema: object | object[] }) {
  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }}
    />
  )
}
```

```tsx
// app/page.tsx (homepage)
import { SchemaOrg } from '@/components/schema-org'

export default function HomePage() {
  const organizationSchema = {
    "@context": "https://schema.org",
    "@type": ["Organization", "LocalBusiness"],
    "name": "Café Canastra",
    "url": "https://cafecanastra.com",
    "logo": "https://cafecanastra.com/logo-canastra.png",
    "description": "Café especial produzido na Serra da Canastra, MG",
    "address": {
      "@type": "PostalAddress",
      "addressLocality": "Serra da Canastra",
      "addressRegion": "MG",
      "addressCountry": "BR"
    },
    "sameAs": [
      "https://instagram.com/cafecanastra",
      "https://facebook.com/cafecanastra"
    ]
  }

  return (
    <>
      <SchemaOrg schema={[organizationSchema, faqSchema, websiteSchema]} />
      {/* resto da homepage */}
    </>
  )
}
```

### Estrutura do FAQPage Schema
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "O que é café especial?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Café especial é aquele que obtém pontuação acima de 80 pontos na escala SCA..."
      }
    }
  ]
}
```
