## Story

**As a** search engine (Google/Perplexity/ChatGPT),
**I want** rich and accurate structured data,
**so that** Cafe Canastra content is understood and cited correctly.

## Acceptance Criteria

1. `Article` schema enriched with: `speakable` (SpeakableSpecification with CSS selectors from `geo_config`), `about` (from `geo_config.entities` where type matches), `mentions`, `dateModified` (from `updated_at`), `wordCount`, `articleSection` (from category)
2. `FAQPage` schema auto-generated when post contains one or more `faq` blocks — merges all FAQ items from all FAQ blocks into a single FAQPage schema
3. `Product` schema auto-generated when post contains `product` blocks — one Product schema per product block with: name, price (offers.price), image, description, url, priceCurrency (BRL)
4. `VideoObject` schema auto-generated when post contains `video` blocks — includes: name (caption), description, thumbnailUrl (poster_url), uploadDate, duration (ISO 8601)
5. `HowTo` schema auto-generated when a `heading` block with text containing "Como" (PT) or "How to" (EN) is immediately followed by a `list` block — extracts steps from list items
6. `ImageObject` schema generated for each `image` block with: url, caption, width, height, alt (as `name`)
7. `BreadcrumbList` schema maintained and improved with 4 levels: Home > Blog > Category > Post Title
8. Meta tags `hreflang` generated for posts with translations: query posts sharing same `translation_group_id`, generate `<link rel="alternate" hreflang="pt" href="...">` for each locale
9. Sitemap.xml (`/api/sitemap.xml`) updated to be fully dynamic: includes all published posts with `<lastmod>` from `updated_at`, `<changefreq>weekly</changefreq>`, `<priority>0.7</priority>`. Excludes drafts and scheduled posts.
10. All schemas generate valid JSON-LD parseable by Google Rich Results Test

## Integration Verification
- IV1: JSON-LD is valid (parseable) on all post pages — no syntax errors
- IV2: No duplicate schemas (one FAQPage per page, one Article per page)
- IV3: Sitemap includes published posts, excludes drafts/scheduled

### Key Technical Details
- **Existing schema generators**: `generateArticleSchema()`, `generateBlogSchema()`, `generateBreadcrumbSchema()` — all in `lib/schemas/blog-schemas.ts`
- **SchemaOrg component**: `components/schema-org.tsx` — renders `<script type="application/ld+json">{JSON.stringify(schema)}</script>`
- **HowTo detection**: Iterate blocks in order. When a heading block contains "Como" (case-insensitive) and the NEXT block is a `list`, generate HowTo schema.
- **FAQPage merging**: If post has 3 FAQ blocks with 2 items each = 1 FAQPage schema with 6 items total.
- **hreflang**: Use Next.js `generateMetadata()` → `alternates.languages` which auto-generates `<link rel="alternate">` tags.
- **Sitemap**: Currently in `app/api/sitemap.xml/route.ts`, returns XML with static URLs. Add dynamic posts query using Supabase anon key (public, status='published' via RLS).
- **Supabase for sitemap**: Use anon key (public read), RLS policy `status = 'published'` handles filtering.
