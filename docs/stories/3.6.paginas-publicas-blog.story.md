## Story
**As a** visitante do site Café Canastra,
**I want** ver uma listagem dos posts do blog em `/blog` e ler posts individuais em `/blog/[slug]` com conteúdo rico e informações de SEO completas,
**so that** eu possa encontrar conteúdo relevante sobre café especial e a Serra da Canastra via busca orgânica.

## Acceptance Criteria

1. A página `/blog` exibe:
   - Grid de cards com os posts publicados (máx 9 por página)
   - Cada card contém: imagem de capa, categoria, título, excerpt, data de publicação e tempo estimado de leitura
   - Ordenação: `published_at DESC`
   - Filtro por categoria (Todos / Receitas / Terroir & Origem / Métodos de Preparo / Notícias / Curiosidades)
   - Paginação ao ter mais de 9 posts
   - Meta tags SEO: title, description, canonical

2. A página `/blog/[slug]` exibe:
   - Imagem de capa (full-width) com alt text correto
   - Título (h1), categoria, data de publicação, autor, tempo de leitura
   - Breadcrumb: Home > Blog > [Categoria] > [Título]
   - Conteúdo do post renderizado (Markdown → HTML com tipografia adequada)
   - Seção "Posts Relacionados" com até 3 posts da mesma categoria
   - Schema markup `Article` e `BreadcrumbList` em JSON-LD
   - Meta tags SEO completas (title, description, OG, canonical)

3. O tempo de leitura é estimado a 200 palavras por minuto e exibido como "X min de leitura".

4. Quando o slug não existe, a página retorna 404 via `notFound()` do Next.js.

5. A listagem do blog exibe mensagem "Nenhum post publicado ainda." quando não há posts.

6. O conteúdo Markdown é renderizado com suporte a: cabeçalhos (h1-h6), negrito, itálico, listas, links, citações (blockquote), e código inline.

7. Todas as imagens dos posts usam o componente `<Image>` do Next.js com `width`, `height` e `alt` definidos.

8. As páginas são renderizadas com Server-Side Rendering (SSR) ou Static Generation com revalidação periódica (`revalidate`) para que novos posts apareçam sem rebuild.

9. A API pública `GET /api/posts` retorna apenas posts com `status = 'published'`, ordenados por `published_at DESC`.

10. O build e lint passam sem erros.

### Rendering de Markdown

```bash

```

```tsx
// Uso na página do post
import ReactMarkdown from 'react-markdown'
import remarkGfm from 'remark-gfm'

<article className="prose prose-amber max-w-none">
  <ReactMarkdown remarkPlugins={[remarkGfm]}>
    {post.content}
  </ReactMarkdown>
</article>
```

### Tempo de Leitura

```typescript
// lib/utils/reading-time.ts
export function estimateReadingTime(content: string): number {
  const wordsPerMinute = 200
  const wordCount = content.trim().split(/\s+/).length
  return Math.ceil(wordCount / wordsPerMinute)
}
```
