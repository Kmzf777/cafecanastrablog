## Story
**As a** Rafael (proprietário do Café Canastra),
**I want** que os posts agendados sejam publicados automaticamente no dia e horário definidos, via script na VPS que roda a cada 5 minutos,
**so that** eu não precise fazer nenhuma ação manual para publicar posts que já agendei.

## Acceptance Criteria

1. Existe uma API endpoint em `POST /api/publish-scheduled` que:
   - Verifica o header `Authorization: Bearer [CRON_SECRET]` (protegido por segredo)
   - Busca todos os posts com `status = 'scheduled'` AND `scheduled_at <= NOW()`
   - Para cada post encontrado: atualiza `status = 'published'` e `published_at = NOW()`
   - Retorna JSON: `{ published: number, posts: string[] }` com contagem e slugs publicados

2. O endpoint retorna 401 se o header de autorização estiver ausente ou incorreto.

3. O endpoint retorna 200 com `{ published: 0, posts: [] }` quando não há posts para publicar (sem erro).

4. Existe um script Node.js em `scripts/publish-scheduled.js` que:
   - Lê `CRON_SECRET` e `NEXT_PUBLIC_SITE_URL` de variáveis de ambiente (arquivo `.env` na raiz)
   - Faz uma requisição POST para `[SITE_URL]/api/publish-scheduled` com o header correto
   - Registra o resultado em log (console + arquivo `logs/cron.log`)
   - Em caso de erro de rede: registra o erro e sai com código 1
   - Usa apenas módulos nativos do Node.js (sem dependências extras — apenas `https`/`http` nativo ou `fetch`)

5. O script pode ser adicionado ao crontab da VPS com o comando:
   ```
   */5 * * * * node /caminho/para/scripts/publish-scheduled.js >> /var/log/cron-blog.log 2>&1
   ```

6. Existe um arquivo `scripts/README-cron.md` com instruções claras de configuração do cron na VPS, incluindo variáveis necessárias e comando de crontab.

7. A variável `CRON_SECRET` é adicionada ao `.env.example` com instrução de preenchimento.

8. O build e lint do projeto passam sem erros.

## Dev Notes

### Endpoint de Publicação Agendada

```typescript
// app/api/publish-scheduled/route.ts
import { createServiceClient } from '@/lib/supabase-service'
import { NextRequest } from 'next/server'

export async function POST(request: NextRequest) {
  // Verificar autorização
  const authHeader = request.headers.get('authorization')
  const expectedAuth = `Bearer ${process.env.CRON_SECRET}`

  if (!authHeader || authHeader !== expectedAuth) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 })
  }

  const supabase = createServiceClient()

  // Buscar posts elegíveis para publicação
  const { data: posts, error } = await supabase
    .from('blog_posts')
    .select('id, slug')
    .eq('status', 'scheduled')
    .lte('scheduled_at', new Date().toISOString())

  if (error) {
    console.error('Error fetching scheduled posts:', error)
    return Response.json({ error: 'Database error' }, { status: 500 })
  }

  if (!posts || posts.length === 0) {
    return Response.json({ published: 0, posts: [] })
  }

  // Publicar cada post
  const ids = posts.map(p => p.id)
  const { error: updateError } = await supabase
    .from('blog_posts')
    .update({ status: 'published', published_at: new Date().toISOString() })
    .in('id', ids)

  if (updateError) {
    console.error('Error publishing posts:', updateError)
    return Response.json({ error: 'Failed to publish' }, { status: 500 })
  }

  const slugs = posts.map(p => p.slug)
  console.log(`[publish-scheduled] Published ${posts.length} posts: ${slugs.join(', ')}`)

  return Response.json({ published: posts.length, posts: slugs })
}
```

### Script Node.js do Cron

```javascript
// scripts/publish-scheduled.js
#!/usr/bin/env node

// Carregar .env (Node.js 20+ tem suporte nativo ou usar dotenv)
const path = require('path')
require('dotenv').config({ path: path.join(__dirname, '../.env') })

const fs = require('fs')
const logsDir = path.join(__dirname, '../logs')
if (!fs.existsSync(logsDir)) fs.mkdirSync(logsDir, { recursive: true })

function log(message) {
  const timestamp = new Date().toISOString()
  const line = `[${timestamp}] ${message}\n`
  process.stdout.write(line)
  fs.appendFileSync(path.join(logsDir, 'cron.log'), line)
}

async function main() {
  const siteUrl = process.env.NEXT_PUBLIC_SITE_URL
  const cronSecret = process.env.CRON_SECRET

  if (!siteUrl || !cronSecret) {
    log('ERROR: NEXT_PUBLIC_SITE_URL ou CRON_SECRET não configurado')
    process.exit(1)
  }

  try {
    const response = await fetch(`${siteUrl}/api/publish-scheduled`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${cronSecret}`,
        'Content-Type': 'application/json',
      },
    })

    if (!response.ok) {
      log(`ERROR: HTTP ${response.status} - ${await response.text()}`)
      process.exit(1)
    }

    const result = await response.json()
    if (result.published > 0) {
      log(`Published ${result.published} posts: ${result.posts.join(', ')}`)
    } else {
      log('No posts to publish.')
    }
  } catch (error) {
    log(`ERROR: ${error.message}`)
    process.exit(1)
  }
}

main()
```
