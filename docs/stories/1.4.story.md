# Story 1.4 — Generate Page (Theme + URL Input)

**Epic:** AI Content Generation Module
**Status:** InProgress
**Priority:** High
**Complexity:** S (3 points)
**Agent:** @dev

---

## Description

Create the frontend page at `/admin/posts/generate` where the user enters a theme and an optional URL to generate an AI-powered blog post. This is the entry point of the AI content generation flow.

## Acceptance Criteria

### AC-1: Page Layout
- [x] Given the user navigates to `/admin/posts/generate`, then the page displays a clean form with: TextArea for theme, Input for URL, and a Generate button
- [x] Given the theme TextArea, then it has placeholder "Ex: Processo de torra do cafe especial na Serra da Canastra"
- [x] Given the theme TextArea, then it enforces min 10, max 500 characters with visible character count
- [x] Given the URL input, then it is clearly marked as optional with helper text
- [x] Given the URL input, then it has placeholder "https://exemplo.com/artigo-sobre-cafe"
- [x] Given the URL input, then it validates URL format on blur

### AC-2: Generate Action
- [x] Given valid theme input, when user clicks "Gerar Post", then it calls `POST /api/admin/ai/generate`
- [x] Given the generation is in progress, then a loading state is shown (spinner + "Gerando seu post..." message)
- [x] Given the generation is in progress, then the Generate button is disabled
- [x] Given generation succeeds, then the user is redirected to `/admin/posts/generate/preview?postId={id}`
- [x] Given generation fails, then an error toast is shown with the error message and a retry option

### AC-3: URL Validation
- [x] Given an invalid URL format, then a validation error is shown below the input
- [x] Given an empty URL field, then no validation error is shown (field is optional)

### AC-4: Authentication
- [x] Given the page, then it requires admin authentication (same as other admin pages)
- [x] Given an unauthenticated user, then they are redirected to `/admin/login`

## Technical Notes
- Follow existing admin page patterns (layout, styling, auth check)
- Use existing UI components: Button, Textarea, Input from shadcn/ui
- Use sonner for toast notifications (already in project)
- Use `useRouter` for navigation after generation
- Loading state should feel responsive — consider showing elapsed time or progress steps
- Page should be `'use client'` directive (client component with form state)
