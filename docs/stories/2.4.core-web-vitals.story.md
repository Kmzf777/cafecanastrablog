## Story
**As a** responsável pelo SEO do Café Canastra,
**I want** que o site tenha boas pontuações nos Core Web Vitals (LCP, FID/INP, CLS) e no Lighthouse,
**so that** o Google considere o site com boa experiência de página (page experience signal), melhorando o ranqueamento orgânico.

## Acceptance Criteria

1. O Lighthouse score em modo desktop para a homepage atinge:
   - Performance: ≥ 90
   - SEO: ≥ 95
   - Accessibility: ≥ 90
   - Best Practices: ≥ 90

2. LCP (Largest Contentful Paint) da homepage é < 2.5 segundos em conexão simulada de 4G.

3. CLS (Cumulative Layout Shift) da homepage é < 0.1.

4. Todas as imagens da homepage que não estão na viewport inicial são carregadas com `loading="lazy"`.

5. A imagem hero (LCP element) é carregada com `priority` (no Next.js Image) ou `fetchpriority="high"` para ser pré-carregada.

6. Fontes externas (Google Fonts ou similares) são carregadas com `font-display: swap` ou substituídas por fontes do sistema.

7. O mapa interativo da fazenda já usa `dynamic()` com SSR desabilitado — confirmar que continua correto e não causa CLS.

8. Nenhuma imagem nova adicionada pelo projeto (blog, admin) é carregada sem dimensões explícitas (para evitar CLS).

9. O build (`npm run build`) passa sem erros ou warnings de performance críticos.

## Dev Notes

### Contexto da Homepage
O arquivo `cafecanastrablog/app/page.tsx` é grande (~41KB, "use client"). Isso implica:
- Toda a homepage é renderizada no cliente
- Pode afetar LCP pois o conteúdo aguarda JS

### Imagens Disponíveis em `/public`
O projeto já possui versões low-quality das imagens (para uso como placeholder/blur):
```
public/degustacao.jpg          + public/degustacao-low.png
public/experiencia-unica.jpg   + public/experiencia-unica-low.png
public/moagem-fresca.jpg       + public/moagem-fresca-low.png
public/plantio-artesanal.jpg   + public/plantio-artesanal-low.png
public/preparo-perfeito.jpg    + public/preparo-perfeito-low.png
public/secagem-solar.jpg       + public/secagem-solar-low.png
public/torra artesanal.jpg     + public/torra-artesanal-low.png
public/banner-cafecanastra.png
public/logo-canastra.png
```

Usar versões `-low.png` como `placeholder="blur"` no Next.js `<Image>` melhora percepção de carregamento.

### Padrão Next.js Image
```tsx
import Image from 'next/image'

// Hero (LCP) — sempre priority
<Image
  src="/degustacao.jpg"
  alt="Degustação de café especial Café Canastra"
  width={1920}
  height={1080}
  priority
  placeholder="blur"
  blurDataURL="/degustacao-low.png"
/>

// Imagens abaixo da dobra — lazy (padrão Next.js)
<Image
  src="/moagem-fresca.jpg"
  alt="Processo de moagem fresca do café"
  width={800}
  height={600}
  // loading="lazy" é o padrão quando priority não está definido
/>
```

### Fontes com Next.js
```tsx
// app/layout.tsx
import { Playfair_Display, Inter } from 'next/font/google'

const playfair = Playfair_Display({
  subsets: ['latin'],
  display: 'swap', // evita FOIT
})
```

### Mapa Interativo
O projeto usa `react-leaflet` com `dynamic()`. Confirmar que está assim em `app/page.tsx`:
```tsx
const FarmMap = dynamic(() => import("@/components/FarmMap"), {
  ssr: false,
  loading: () => (
    <div
      className="w-full h-full bg-amber-50 animate-pulse"
      style={{ minHeight: '400px' }} // evita CLS
    />
  )
})
```
