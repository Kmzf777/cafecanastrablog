## Story

**As a** blog administrator,
**I want** to see the complete status of all posts with SEO metrics,
**so that** I can manage content efficiently.

## Acceptance Criteria

1. Dashboard displays post list with columns: Title, Status (colored badge), Category, Locale (flag icon), Date (published or scheduled or created), SEO Score (colored badge), Actions
2. Status badges: Published = green, Scheduled = yellow, Draft = gray
3. SEO score badges: green (90+), yellow (70-89), orange (50-69), red (<50), gray (N/A for posts without score)
4. Locale flags: PT = Brazilian flag emoji, EN = US/UK flag emoji, ES = Spanish flag emoji
5. Filters: status dropdown (All/Published/Scheduled/Draft), category dropdown (All + existing categories), locale dropdown (All/PT/EN/ES)
6. Search by title (client-side filter, debounced 300ms)
7. Sort by: Date (default, desc), Title (asc), SEO Score (desc) — clickable column headers
8. Actions per post: Edit (always), Preview (if published — opens `/blog/{slug}` in new tab), Duplicate (creates copy as draft), Delete (with confirmation dialog)
9. "New Post" button in header directing to the block editor (`/admin/posts/new`)
10. Summary counters at top: Published: X | Draft: Y | Scheduled: Z

## Integration Verification

### Key Technical Details
- **Existing dashboard**: `app/admin/page.tsx` currently lists posts with basic info and status badges. This story enhances it significantly.
- **Supabase query**: Fetch from `blog_posts` using service role key. Include `seo_score`, `locale`, `category`. Don't JOIN blocks for listing (performance).
- **Client-side filtering**: After initial fetch, filters and search operate on the fetched array. No additional API calls for filtering.
- **Duplicate endpoint**: Copy post metadata (new UUID, new slug with `-copy` suffix), copy all blocks with new UUIDs, set status to draft, clear published_at/scheduled_at.
- **Delete**: Use Supabase `.delete()`. The `ON DELETE CASCADE` on `blog_post_blocks.post_id` handles block cleanup automatically.
- **Flag emojis**: Use Unicode flags or simple text labels if rendering is inconsistent.

- **Story 1.2** — API endpoints for posts
