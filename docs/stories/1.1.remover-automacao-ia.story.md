## Story
**As a** desenvolvedor do projeto,
**I want** remover completamente o sistema de automação de blog via IA (web scraping + OpenAI/Gemini) do codebase,
**so that** o projeto fique limpo de código não testado, as dependências desnecessárias sejam removidas, e o build passe sem erros — preparando a base para o novo sistema de blog.

## Acceptance Criteria

1. Os seguintes arquivos são completamente removidos do projeto:
   - `cafecanastrablog/services/blogGenerationService.ts`
   - `cafecanastrablog/services/scraperService.ts`
   - `cafecanastrablog/services/openaiService.ts`
   - `cafecanastrablog/app/api/automacaoblog/route.ts` (e pasta `automacaoblog/`)
   - `cafecanastrablog/app/cafecanastrablog/page.tsx`
   - `cafecanastrablog/app/cafecanastra/layout.tsx` (se usado exclusivamente pela automação)
   - `cafecanastrablog/scripts/scheduled-posts.js`
   - `cafecanastrablog/scripts/agendamento-aleatorio.js`
   - `cafecanastrablog/agendamento-posts.bat`
   - `cafecanastrablog/database/extend_blog_posts_for_automation.sql`

2. As seguintes dependências npm são removidas do `package.json` (apenas se não utilizadas em nenhuma outra parte do projeto):
   - `@mozilla/readability`
   - `jsdom`
   - `cheerio`
   - `openai`
   - `@google/genai`

3. O comando `npm run build` executa sem erros após todas as remoções.

4. O comando `npm run lint` executa sem erros ou warnings relacionados aos arquivos removidos.

5. Nenhuma importação quebrada ou referência órfã a arquivos removidos permanece no codebase (verificado via lint e build).

6. O arquivo `automacaoblog/pdr.md` na raiz do projeto é removido ou arquivado (não afeta o build, mas é limpeza).

7. O middleware existente (`cafecanastrablog/middleware.ts`) não possui mais referências a rotas da automação removida.
