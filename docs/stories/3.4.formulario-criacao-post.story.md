## Story
**As a** Rafael (proprietário do Café Canastra),
**I want** um formulário completo para criar e editar posts do blog com título, conteúdo, imagem de capa, data/hora de publicação e categoria,
**so that** eu consiga escrever e agendar posts de forma simples, sem precisar de conhecimento técnico.

## Acceptance Criteria

1. A página `/admin/posts/new` exibe um formulário com os seguintes campos:
   - **Título** — input de texto, obrigatório, máx 80 chars, com contagem de caracteres
   - **Slug** — gerado automaticamente a partir do título (editável), exibido abaixo do título
   - **Categoria** — select com opções: Receitas / Terroir & Origem / Métodos de Preparo / Notícias / Curiosidades
   - **Conteúdo** — textarea ampla, obrigatório, suporte a Markdown (texto puro, renderizado na página pública)
   - **Excerpt** — textarea curta, opcional, máx 300 chars (gerado dos primeiros 200 chars do conteúdo se vazio)
   - **Imagem de capa** — upload de arquivo (JPEG/PNG/WebP, máx 5MB) com preview
   - **Alt da imagem** — input de texto, descritivo para SEO e acessibilidade
   - **Data e hora de publicação** — datetime-local input, opcional
   - **Tags** — input com múltiplas tags (separadas por Enter ou vírgula)

2. O slug é gerado automaticamente ao digitar o título:
   - Minúsculo
   - Sem acentos (ã→a, é→e, etc.)
   - Espaços substituídos por hífens
   - Sem caracteres especiais
   - Editável manualmente pelo usuário

3. O upload de imagem:
   - Valida tamanho máximo de 5MB (erro se excedido)
   - Valida tipo de arquivo (apenas JPEG/PNG/WebP)
   - Exibe preview da imagem selecionada antes do upload
   - Faz upload para o bucket `blog-images` do Supabase Storage via `/api/admin/upload`
   - Exibe indicador de progresso/loading durante upload

4. Existem três botões de ação:
   - **"Salvar Rascunho"** — salva com `status: 'draft'`, sem data
   - **"Salvar e Agendar"** — requer data/hora preenchida; salva com `status: 'scheduled'`; erro se data estiver no passado
   - **"Publicar Agora"** — salva com `status: 'published'` e `published_at: NOW()`

5. A página `/admin/posts/[id]/edit` carrega os dados do post existente e permite editar qualquer campo, com os mesmos botões de ação.

6. Após salvar com sucesso, o usuário é redirecionado para `/admin` com toast de confirmação.

7. Validação client-side exibe erros inline (abaixo do campo) para:
   - Título vazio
   - Conteúdo vazio
   - Data no passado ao tentar agendar

8. A API `/api/admin/upload` recebe um arquivo via `multipart/form-data`, faz upload para Supabase Storage e retorna a URL pública da imagem.

9. As APIs de criação e atualização (`POST /api/admin/posts` e `PUT /api/admin/posts/[id]`) verificam autenticação e validam os dados.

10. O build e lint passam sem erros.

### Upload para Supabase Storage

```typescript
// app/api/admin/upload/route.ts
export async function POST(request: Request) {
  await verifyAdminAuth()

  const formData = await request.formData()
  const file = formData.get('file') as File

  // Validação server-side
  if (file.size > 5 * 1024 * 1024) {
    return Response.json({ error: 'Arquivo muito grande (máx 5MB)' }, { status: 400 })
  }

  const allowedTypes = ['image/jpeg', 'image/png', 'image/webp']
  if (!allowedTypes.includes(file.type)) {
    return Response.json({ error: 'Tipo de arquivo não permitido' }, { status: 400 })
  }

  const { createServiceClient } = await import('@/lib/supabase-service')
  const supabase = createServiceClient()

  const filename = `${Date.now()}-${file.name.replace(/[^a-z0-9.-]/gi, '-')}`
  const { error } = await supabase.storage
    .from('blog-images')
    .upload(filename, file, { contentType: file.type })

  if (error) return Response.json({ error: error.message }, { status: 500 })

  const { data } = supabase.storage.from('blog-images').getPublicUrl(filename)
  return Response.json({ url: data.publicUrl })
}
```

### Lógica dos Três Botões de Ação

```typescript
const onSaveDraft = handleSubmit((data) => savePost({ ...data, status: 'draft', scheduled_at: null }))

const onSchedule = handleSubmit((data) => {
  if (!data.scheduled_at) return setError('scheduled_at', { message: 'Data obrigatória para agendar' })
  if (new Date(data.scheduled_at) <= new Date()) return setError('scheduled_at', { message: 'Data deve ser no futuro' })
  savePost({ ...data, status: 'scheduled' })
})

const onPublishNow = handleSubmit((data) => savePost({ ...data, status: 'published', published_at: new Date().toISOString() }))
```
