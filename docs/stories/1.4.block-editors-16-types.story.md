## Story

**As a** content editor,
**I want** specialized editors for each block type,
**so that** I can create rich content without writing Markdown.

## Acceptance Criteria

1. `HeadingEditor` — Level selector (H1-H6 dropdown), text input, `anchor_id` auto-generated from text via slugify
2. `ParagraphEditor` — Rich text editor with TipTap (`@tiptap/react` + `@tiptap/starter-kit` + `@tiptap/extension-link` + `@tiptap/extension-underline` + `@tiptap/extension-highlight`). `InlineToolbar.tsx` appears on text selection with: Bold, Italic, Underline, Link, Highlight, Clear Formatting
3. `ImageEditor` — Upload zone (click or drag file), image preview after upload, required `alt` input (shows warning if empty), optional `caption` input. Uses existing `/api/admin/upload` endpoint
4. `GalleryEditor` — Multi-image upload, layout selector (grid/carousel/masonry), columns selector (2/3/4), drag-to-reorder images within gallery, remove individual image
5. `QuoteEditor` — Textarea for quote text, input for author, input for source_url (labeled "Source URL (recommended for GEO)")
6. `ListEditor` — Toggle ordered/unordered, dynamic item inputs (add/remove), drag-to-reorder items within list
7. `CodeEditor` — Monospace textarea for code, language selector dropdown (javascript, typescript, python, sql, bash, json, html, css, other), optional filename input
8. `EmbedEditor` — URL input with auto-detection of embed type (YouTube, Instagram, Twitter/X, TikTok, generic), preview of embed, caption input
9. `DividerEditor` — Style selector with visual previews (line, dots, space)
10. `CalloutEditor` — Type selector (info/warning/tip/success) with colored preview, rich textarea, optional `source_url` for stat citations
11. `TableEditor` — Editable grid with add/remove row and column buttons, caption input, minimum 2x2
12. `FaqEditor` — Question/Answer pairs with add/remove pair button, reorder pairs, minimum 1 pair
13. `CtaEditor` — Inputs for text, button_text, button_url, style selector (primary/secondary/outline) with visual preview
14. `VideoEditor` — URL input, poster image upload (optional), caption input, duration input (optional, format HH:MM:SS)
15. `AccordionEditor` — Title/content pairs with add/remove, rich text in content, reorder items
16. `ProductEditor` — Inputs for name, price (with currency BRL default), image upload, link URL, description textarea
17. Every editor validates its data against the corresponding Zod schema from Story 1.1 and shows inline validation errors
18. Every editor calls `onUpdate(blockId, data)` on change to sync with BlockEditor state

## Integration Verification
- IV1: Each editor validates its data with the corresponding Zod schema from `block-validation.ts`
- IV2: Editor data serializes correctly to the JSONB format expected by the API
- IV3: All editors are keyboard-accessible (Tab navigation, form inputs focusable)

### Key Technical Details
- **TipTap**: Headless rich text editor based on ProseMirror. Use `useEditor` hook. Serialize output to HTML via `editor.getHTML()`. Store HTML string in `data.text`.
- **Image upload**: Reuse existing `uploadImage` pattern from `PostForm.tsx` — POST to `/api/admin/upload`, returns `{ url: string }`.
- **Embed type detection**: Use regex on URL to detect type:
  - YouTube: `youtube.com/watch` or `youtu.be/`
  - Instagram: `instagram.com/p/` or `instagram.com/reel/`
  - Twitter/X: `twitter.com/` or `x.com/`
  - TikTok: `tiktok.com/@`
  - Default: `generic`
- **Anchor ID**: For HeadingEditor, auto-generate from text using `slugify` (already in project dependencies)
- **Editor prop pattern**: Every editor receives `{ data: BlockData, onChange: (data: BlockData) => void }` — call `onChange` on every field change
- **Existing UI**: Use `Button`, `Input`, `Textarea`, `Select`, `Label` from `components/ui/`
