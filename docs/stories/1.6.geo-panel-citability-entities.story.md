## Story

**As a** content strategist,
**I want** to optimize posts for AI search engines,
**so that** Cafe Canastra is cited as a source in AI-generated answers.

## Acceptance Criteria

1. `geo-analyzer.ts` implemented in `lib/seo/` with metrics:
   - **Citability score**: % of paragraphs that are self-contained (< 3 sentences per paragraph = citable). Target >= 80%
   - **Entity count**: number of semantic entities linked in `geo_config.entities[]`
   - **FAQ coverage**: number of FAQ blocks in the post (recommended >= 1)
   - **Source citations**: count of blocks with `source_url` filled (quote, callout). Recommended >= 2
   - **Speakable coverage**: % of content marked as speakable via selectors
   - **Content freshness**: days since last update
2. `GeoPanel.tsx` implemented as a tab in the editor sidebar displaying all 6 metrics with visual indicators (green/yellow/red)
3. `EntitySelector.tsx` — interface for adding/editing/removing semantic entities with fields: `name`, `type` (dropdown: Thing, Place, Product, Organization, Person), `sameAs` (URL, optional), `description` (optional)
4. Entity list displayed with edit/delete per entity and "Add Entity" button
5. Contextual recommendations displayed based on current content:
   - If FAQ blocks = 0: "Add an FAQ block to improve citability in AI answers"
   - If source citations < 2: "Add source URLs to your quotes and callouts for better authority signals"
   - If citability < 70%: "Break long paragraphs into shorter, self-contained statements"
   - If entities < 2: "Link semantic entities to help AI systems understand your content"
6. Geo targeting fields: `target_region`, `target_city`, `coordinates` (lat/lng inputs)
7. Speakable selectors configuration: checkboxes to include title, excerpt, FAQ answers in speakable markup
8. All GEO data saved to `geo_config` JSONB field via parent form

## Integration Verification
- IV1: GEO analysis is client-side, no external API dependencies
- IV2: Entities saved correctly to `geo_config.entities[]` in JSONB
- IV3: Metrics update in real-time as content changes

### Key Technical Details
- **Sentence counting for citability**: Split paragraphs on `.` `!` `?` followed by space or end. Paragraph with <= 3 sentences = citable.
- **Only analyze `paragraph` blocks** for citability (headings, quotes etc. are inherently structured).
- **Entity `sameAs`**: Should be a Wikidata or Wikipedia URL (e.g., `https://www.wikidata.org/wiki/Q123`). Not validated strictly — just a URL input.
- **Speakable selectors**: CSS selectors that will be used in JSON-LD `SpeakableSpecification`. Map:
  - "Include post title" → `.post-title`
  - "Include excerpt" → `.post-excerpt`
  - "Include FAQ answers" → `.faq-answer`
- **Default geo**: Pre-fill from `SITE_CONFIG.address`: region="Serra da Canastra", city="Medeiros, MG"
- **Content freshness**: Show as "Updated X days ago" with green (<7 days), yellow (7-30), red (>30)
